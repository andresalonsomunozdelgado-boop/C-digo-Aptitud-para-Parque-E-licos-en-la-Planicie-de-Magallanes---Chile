# Código Aptitud para Parque Eólicos en la Planicie de Magallanes Chile
Código que muestra el trabajo completo del análisis multicriterio, junto a sus cambios en la metodología de estudio.

Ante las restricciones impuestas por la memoria volátil (RAM) del entorno de ejecución en la nube, se adoptó una arquitectura de software descentralizada mediante la fragmentación del pipeline analítico en cinco módulos o notebooks secuenciales. Esta estrategia de optimización permitió mitigar el riesgo de desbordamiento de memoria al procesar Big Data geoespacial. Geográficamente, el flujo metodológico operó bajo una lógica de reducción de escala multitemporal y multiespacial: la fase inicial ejecutó la conexión mediante la API de Google Earth Engine (GEE) para la adquisición y procesamiento continuo de variables climáticas a escala nacional. Posteriormente, el área de estudio se delimitó a nivel político-administrativo en la Región de Magallanes, culminando con una focalización estrictamente territorial sobre la Planicie Magallánica y Tierra del Fuego. Este recorte final responde a criterios de optimización técnico-económica, ya que el relieve plano maximiza el factor de capacidad de los aerogeneradores y minimiza los costos constructivos y viales de infraestructura.
